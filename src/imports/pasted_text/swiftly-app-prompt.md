## **Figma AI Design Prompt: Swiftly Mobile App**

Design a complete mobile application for Swiftly, a Grab-like platform connecting homeowners with verified contractors for house-related problems. The app should follow modern mobile UI/UX principles with a clean, professional aesthetic that balances trustworthiness with ease of use. The visual style should utilize a primary purple gradient defined as hex color 667eea transitioning to 764ba2 for all primary branding elements, buttons, and key accents. The color palette should include white (ffffff) for primary backgrounds, light gray (f8f9fa) for secondary backgrounds and cards, medium gray (6c757d) for secondary text and disabled states, dark gray (333333) for primary text, success green (28a745) for confirmation states, warning yellow (ffc107) for urgency indicators, and danger red (dc3545) for error states. Use modern sans-serif typography such as Inter or SF Pro, with font weights ranging from 400 for body text to 700 for headlines, maintaining proper hierarchy and readability.

The app structure should begin with an onboarding flow featuring a welcome screen displaying the Swiftly logo with the tagline "Fast. Reliable. Professional." followed by a tabbed authentication interface allowing users to switch between login and registration modes. The login form should include email and password fields with modern input styling, while the registration form should capture full name, email, phone number, and password confirmation. Include social login options for Google and Apple authentication with their respective brand colors, and implement phone verification as an alternative authentication method with proper validation states.

The main navigation should utilize a bottom tab bar with five primary sections: Home, Categories, Job Requests, Messages, and Profile. The Categories screen should display a responsive grid layout of eight service categories, each featuring a specific icon and label. The categories include House (represented by a house emoji), Condo (building emoji), Commercial (store emoji), Lot (construction emoji), Plumbing (wrench emoji), Electrical (lightning bolt emoji), HVAC (snowflake emoji), and Renovation (hammer emoji). Each category card should have a subtle hover state with a gentle lift animation and border highlight in the primary purple gradient color.

The problem description flow should be implemented as a multi-step form with a progress indicator at the top showing completion percentage. The form should dynamically populate subcategory options based on the selected main category, include a large text area for detailed problem descriptions with character count guidance, and feature an urgency selector with four levels color-coded from green (low priority) to red (emergency). The budget selection should use visual range indicators with clear price bands, and the location input should include a map integration placeholder with current location detection. Include a photo upload section allowing users to add up to five images of their problem area.

The contractor discovery interface should present contractors in a clean list format with cards displaying profile photos or generated initials on a purple gradient background, contractor names and business names, specialty descriptions, five-star rating systems with decimal precision, response time indicators, and availability status badges. Implement comprehensive filtering options for price range, availability, and rating minimums, along with sorting options by rating, response time, or distance. Include a toggle between list view and map view with location-based contractor clustering.

Contractor profile screens should feature a hero section with the contractor's photo, name, business name, and key statistics prominently displayed. Below this, include a detailed ratings section showing overall score with star visualization and individual category ratings, a reviews feed with user photos and timestamps, a services section with pricing indicators, and a portfolio gallery with before/after project images. Display verification badges for licensing, insurance, and background checks using checkmark icons in green circles. The primary call-to-action should be "Request Quote" in the brand purple gradient, avoiding immediate hiring actions.

The job requests dashboard should organize user requests into three tabs: Active Requests, Past Jobs, and Drafts. Each job card should display the category, subcategory, description preview, urgency level with color coding, budget range, posting date, and a status indicator using specific emojis: yellow circle for waiting for responses, green circle for responses received, blue circle for contractor selected, black circle for in progress, and checkmark for completed. Implement a response comparison view that allows side-by-side evaluation of multiple contractor proposals with pricing, timeline, and detailed responses.

The response management interface should display contractor quotes in card format with the proposed amount prominently displayed at the top in large, bold text, followed by estimated timeline information, detailed proposal text, and a preview of the contractor's profile with rating and photo. Each response card should contain two primary action buttons: "Select Contractor" in the brand purple gradient and "View Profile" as an outlined button. Include a comparison mode that stacks multiple responses vertically with aligned pricing and timeline information for easy evaluation.

The messaging system placeholder should feature a modern chat interface with message bubbles differentiated by sender, typing indicators with animated dots, read receipt checkmarks, and file/photo sharing capabilities. Include quick reply templates for common responses and the ability to share quotes directly within the conversation thread.

Navigation patterns should follow mobile best practices with swipe gestures for card actions like archiving or deleting, pull-to-refresh functionality on all list screens, and a clear back navigation system with proper screen hierarchy. Implement micro-interactions including smooth screen transitions with slide animations, skeleton loading states for data fetching, success animations with checkmarks and confetti effects for completed actions, and haptic feedback for important interactions like confirming selections.

The design system should be built on an 8-point grid for consistent spacing, with minimum touch targets of 44x44 pixels for all interactive elements. Component libraries should include reusable button variants (primary, secondary, outlined, danger), card styles for different content types, input field variations, badge styles for status indicators, and icon sets for all service categories and actions. Ensure accessibility compliance with WCAG 2.1 AA standards, proper color contrast ratios, screen reader labels, and support for large text sizes and voice control.

The app should support both light and dark themes, with the dark theme using a deep navy background (1a1a2e) instead of pure black, maintaining the purple gradient accents and ensuring all text remains readable with proper contrast ratios. Include onboarding illustrations that show diverse homeowners and contractors in various home improvement scenarios, maintaining a professional yet approachable visual style throughout the application.

The final deliverable should include a complete mobile app UI design with all screens and states, a comprehensive component library with usage guidelines, user flow diagrams showing the complete job request and hiring process, an interactive prototype demonstrating key interactions, and detailed design system documentation including typography scales, color usage rules, and spacing guidelines. The design should prioritize trust-building elements like verification badges, transparent pricing, and clear status indicators while maintaining an efficient, intuitive user experience that minimizes friction in the job request and contractor selection process.
